What Is the x4 Model?

The x4 Model is a client-side encryption framework that applies four sequential layers of AES‑GCM encryption, each derived through HKDF (HMAC-based Key Derivation Function).

Four-Layer Security

Four sequential layers of AES-GCM encryption with unique salts and IVs for each sub-layer guarantee that even if one key is compromised, three more barriers remain intact.

Client-Side Encryption

All encryption and decryption happen entirely within the browser; servers never see plaintext or raw keys, ensuring maximum privacy.

Dual Password Protection

Two separate passwords are required, each undergoing a two-step HKDF → AES‑GCM process, doubling the encryption depth.

Four Layers of Security

Each step re-randomizes salts/IVs, yielding ciphertexts that are unique, non-malleable, and resistant to side‑channel or rainbow‑table attacks.

Layer 1 & 2 (User Password 1)

The first two layers of encryption use the first password to create a double-layered protection system.

1

HKDF Key Derivation

The first HKDF derivation salts and stretches the user's primary password into a 256-bit AES‑GCM key.

2

First Layer Encryption

A unique salt + IV is generated for the first layer, encrypting the plaintext.

3

Second Layer Encryption

The second derivation uses a new salt and context string, protecting even the intermediate ciphertext.

Layer 3 & 4 (User Password 2)

The third and fourth layers add another double-layer of protection using a completely separate password.

1

HKDF Key Derivation

Password 2 undergoes the same two-step HKDF → AES‑GCM process, doubling the encryption depth.

2

Third Layer Encryption

The third layer applies encryption to the already twice-encrypted data.

3

Fourth Layer Encryption

Separate salts and IVs for each sub-layer guarantee that even if one key is compromised, three more barriers remain intact.

What It Does

The x4 Model provides powerful encryption capabilities with a simple, user-friendly interface.

Encrypt

Users supply two passwords and a plaintext message. x4 performs four sequential encryptions client-side, outputting a Base64 string.

Decrypt

Users supply the same two passwords and the Base64 ciphertext. x4 reverses the four layers, revealing the original plaintext if—and only if—both passwords match exactly.

Copy & Share

One-click copy to clipboard with visual feedback. Encrypted messages can be safely shared over any channel—email, messaging apps, cloud storage—without leakage.

Frequently Asked Questions

Get answers to common questions about the x4 Model encryption system.

Can I recover my message if I forget one password?

No. Both passwords are required in the exact order. x4 cannot decrypt without complete key material.

Is my data stored anywhere?

Only in your clipboard or local storage if you choose. x4 never sends plaintext or keys over the network.

Which browsers are supported?

All modern browsers with the Web Crypto API (Chrome, Firefox, Safari, Edge). Older browsers will fall back gracefully.

How secure is the x4 Model?

The x4 Model provides extremely high security through:

  • Brute Force Resistance: Two passwords stretched via PBKDF2/HKDF → 4 layers of AES-GCM raises the cost of any attack exponentially.
  • Forward Secrecy (Ephemeral Salts): Unique per-message salts/IVs prevent replay or rainbow-table attacks.
  • Integrity Checks: AES-GCM's built-in authentication tag detects any tampering.
