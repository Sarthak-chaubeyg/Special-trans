How x4 Encryption Works

Step-by-step breakdown of the encryption and security process

01

Input & Instant Clearing

Passwords are read from DOM and immediately cleared from input fields. Values are converted to Uint8Array for processing.

DOM ‚Üí Read ‚Üí Clear (1ms) ‚Üí Uint8Array
02

PBKDF2 Key Derivation

Each password undergoes PBKDF2 with domain-separated salts. Iterations calibrated for ~300ms derivation time.

PBKDF2(password, salt|context, 600K iterations)
03

Four-Layer Encryption

Sequential AES-256-GCM encryption with unique parameters per layer. AAD binding ensures tamper detection.

L1(p1) ‚Üí L2(p1) ‚Üí L3(p2) ‚Üí L4(p2)
04

Memory Sanitization

All password bytes are immediately zeroed. Buffer references cleared. String variables reset to empty.

passwordBytes.fill(0) ‚Üí buffer.slice() ‚Üí GC
05

Output with Header

Self-describing header includes all parameters. Base64 URL-safe encoding for universal compatibility.

x4v2|params||base64url(ciphertext)
